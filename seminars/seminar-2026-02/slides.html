Software Development vs Data Analysis: Different AI Roles

Software Development

  • Goal: Build functional systems
  • Success metric: Binary (works/breaks)
  • Central question: ‚ÄúDoes it execute correctly?‚Äù
  • AI‚Äôs strength:
    • Generate boilerplate code
    • Suggest syntax and patterns
    • Automate testing
    • Refactor code structure
  • Primary risk: Bugs, crashes, security flaws
  • Verification: Unit tests, integration tests
  • Trust level: High for implementation

Data Analysis

  • Goal: Generate valid insights
  • Success metric:
  • Central question: ‚ÄúAre these conclusions sound?‚Äù
  • AI‚Äôs strength:
    • Write data manipulation code
    • Create visualizations
    • Suggest statistical methods
    • Automate repetitive tasks
  • Primary risk: Correct code producing invalid conclusions
  • Verification: Domain expertise, theoretical validity
  • Trust level: Low for interpretation

Warning

Key insight: In analysis, syntactically correct code ‚â† methodologically sound results. AI can produce perfect code that violates logic and common sense, or draws invalid inferences.

Note

Bottom line: AI excels at the mechanics of analysis (data cleaning, computation, visualization). Researchers must retain ownership of scientific judgment: choosing appropriate methods, validating assumptions, and interpreting substantive meaning.

Software Setup Overview

Note

Full details: Setup Instructions

  1. Install prerequisite software (via WB Software Center)
    • Stata 19+, R 4.5+, Python 3.13+, Quarto, Git
    • Install Python uv package: pip install uv
  2. Install Positron IDE (system-level install): Request help from IT if needed
  3. Install key extensions in Positron
    • Stata MCP, Quarto
  4. Connect GitHub: Sign in to GitHub account
    • Enable Positron Assistant in settings
    • Configure LLM provider (GitHub Copilot/Claude/GPT)
  5. Start experimenting!
    • Open assistant: Ctrl+Shift+P > ‚ÄúAsk Positron Assistant‚Äù
    • Try: chat, agent mode, inline code completion
