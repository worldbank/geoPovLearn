Example 1: From Messy Code to Reproducible Package

Live demonstration of building a reproducibility package with AI assistance

Example 1: Summary

  • Integrated Development Environment (IDE) - Positron

  • GitHub Copilot that uses Anthropic Claude Sonnet 4.5 LLM to provide

    • Inline code completions
    • Chat: help and assistance
    • Agent: active changes to my code and code execution

  • Under 5 mins we’ve created a well-structured reproducibility package from messy code running both: Stata and R and creating fully customized graphics and reports.

How does this different from traditional AI chat?

  • We did not copy-paste anything from chat windows to our code.

  • The AI agent was fully aware of our project context, files, data, and code.

  • The AI agent was able to autonomously execute code, make changes to files, and create new files.

  • The AI agent was integrated into our development environment (IDE) and worked alongside us.

  • The AI agent did not have access to our data outside of our local machine and IDE.

    • All code was executed locally on our machine.

AI-Chat vs AI-Agent vs Inline Code Completion (1)

AI-Chat: Ask Mode

  • Conversational interface
  • General purpose assistance
  • Limited context awareness: knows only what you provide
  • One‑off questions
  • Code snippets
  • Explanations of code/output

. . .

Usage

Use Ask mode to think.

AI-Chat vs AI-Agent vs Inline Code Completion (2)

AI-Agent: Execute Mode

  • Autonomous task execution
  • Full project context awareness
  • Integrated with development tools
  • Can read/write files, run code, and make changes (after user approval)
  • Multi‑step workflows
  • End‑to‑end tasks (clean → estimate → table → robustness)

. . .

Usage

Use Agent mode to execute explicit instructions

AI-Chat vs AI-Agent vs Inline Code Completion (3)

Inline Code Completion

  • Real-time code suggestions while you type
  • Can be context-aware or not
  • Speeds up coding by autocompleting:
    • Function names
    • Parameters
    • Common patterns

Context Awareness

What AI “knows” when helping you code:

  • Your files: All code files, documentation, and project structure
  • Your data structure: Variable names, data types, relationships (without sending raw data to cloud)
  • Your history: Recent edits, commands run, and conversations
  • Your environment: Installed packages, dependencies, and configurations
  • Your intent: Current task based on files you have open and recent actions
  • Results of code execution: Output, errors, and logs from running code

Why it matters:

  • AI gives relevant suggestions specific to your project
  • No need to copy-paste code context manually
  • AI understands dependencies between files
  • Reduces hallucinations by grounding responses in your actual codebase

Model Context Protocol (MCP)

“…a standardized, open framework for AI models (LLMs) to securely and efficiently connect with external applications, data sources, tools, and each-other — acting like a universal adapter (e.g., USB-C for AI)”

— Google AI Search + Wikipedia

Essential connections:

  • Execution environments: Stata, R, Python
  • Editors: Positron, VS Code, RStudio, JupyterLab

Integrated Development Environments (IDEs)

Your coding workspace:

For Data Science:

  • Positron (Posit)
  • RStudio
  • JupyterLab
  • VS Code

Key Features:

  • Code editor
  • Console/terminal
  • File browser
  • Variable viewer
  • Plot viewer

LLMs for Coding

Large Language Models powering AI assistants:

  • GitHub Copilot: Uses OpenAI models (GPT-4) and Anthropic Claude
  • OpenAI GPT: GPT-4, GPT-4 Turbo
  • Anthropic Claude: Opus, Sonnet, Haiku
  • Others: Google Gemini, Meta Llama, Mistral

Software Development vs Data Analysis

While these tools were built for software development, they excel at data analysis tasks: data cleaning, statistical modeling, visualization, and report generation.
